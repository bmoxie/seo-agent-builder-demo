Six-stage operating loop

Individual agents also map to an end-to-end loop: Sense → Decide → Produce → Orchestrate → Verify → Learn. Three substrates sit under every stage: Marketing Brain (shared memory), Governance Gate (policy on recommend / draft / route / execute), and Ownership Registry (who owns what). The Workflows page lists which named agents roll up into each stage and how the end-to-end examples plug into that loop.

How to read this diagram

Top down = how a request flows. A team member asks something in Teams or Outlook (Layer 2). That triggers an agent (Layer 3). The agent consults the Marketing Brain and Governance Gate (Layer 4) to stay on-message and within policy. It calls the right tools through MCPs (Layer 5), reads from the unified data lake (Layer 6), which is fed nightly by external APIs (Layer 7). All of it runs on Azure infrastructure that the developers maintain in Cursor and Git (Layer 8).

Priority dots (next to each agent): green=P1 · orange=P2 · yellow=P3 · red=P4 · slate=other. In bucket 1, combined lines inherit the strongest brainstorm match: e.g. Field News & Macro Impact uses the P1 dot (Field News TLDR was P1; Macro was P2); Content Gap & FAQ Cannibalization uses P3 (FAQ + Cannibalization was P3; Content Gap Analysis was P4).

Bucket 8 also lists Marketing Brain, Governance Gate, and Ownership Registry as operational touchpoints — the same capabilities appear in Layer 4 as shared infrastructure every agent consults.

The big idea: the team supervises the system instead of doing every task by hand. The Marketing Brain is the multiplier — every agent gets smarter the day it ships because they all share one source of company knowledge.

What's custom vs. off-the-shelf: Only six MCPs are custom-built (Unified Data Lake, Marketing Brain, Governance Gate, Ownership Registry, AEM, Profound). Everything else is either an off-the-shelf connector or an API feeding the data lake on a schedule.
