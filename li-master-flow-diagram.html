How work moves through the system

Solid gray arrows: requests, routing, tools, and data. Purple dash: policy with agents. Light dotted ring: end-to-end loop. External feeds (lower left) load the lake beside the main path. Specialist agents group into eight buckets—full roster in Architecture.

Operating cycle

Work runs as a continuous loop, not a sequence of handoffs. Learn feeds memory and prioritization; the next Sense picks up in a smarter context.

Sense→ Decide→ Produce→ Orchestrate→ Verify→ Learn→ ↻ Sense…

Always-on underneath every turn of the loop

  • Marketing Brain — shared memory each part of the loop reads and writes
  • Governance Gate — recommend · draft · route · execute on production and routing
  • Ownership Registry — who owns URL, prompt family, and BU

Which agents map to each part of the loop, plus five example diagrams (slices through Sense→Learn): Workflows.

Entry

Humans and partners open work through Microsoft surfaces and line-of-business tools; the orchestrator turns chat, email, and backlog signals into routed work.

Core loop

Agents execute with MCPs against the data layer; brain and governance sit on the execution path so recommendations and drafts stay on-brand and policy-safe.

Learn & return

External feeds hydrate the lake; readouts and dashboards close the loop so stakeholders see outcomes where they already work.
